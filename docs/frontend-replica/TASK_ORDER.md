# FRONTEND-REPLICA Task Order

| Task | Name | Parallelization | Output |
| --- | --- | --- | --- |
| R01 | 设计清单完善 | 可与 R02/R03 并行 | 补人工 canonical、状态、模块、裁切候选；确认 P0/P1/P2。 |

| R03 | 视觉 token 抽取 | 可与 R01/R02 并行 | 抽色彩、圆角、阴影、字体、间距、图标风格和响应式断点。 |
| R04 | 组件基建 | 依赖 R03 | 沉淀 shell、nav、cards、charts、assistant panel、drawer/modal 复用层。 |
| R05 | 图表复刻 | 依赖 R03/R04，可与 R10-R40 并行 | 复刻 KPI、折线、柱状、环图、组合图、运营报表。 |
| R06 | 三端 AI 助手复刻 | 依赖 R03/R04，可与 R10-R40 并行 | 园长/教师/家长 AI 工作区 UI 与 vivo 调用边界。 |
| R10 | 登录与注册 | 依赖 R04，可与 R20/R30/R40 并行 | 登录、注册弹窗、示例账号、移动端。 |
| R20 | 园长端 | 依赖 R04，可并行 | 首页、AI、周报、运营报表、教师管理。 |
| R30 | 教师端 | 依赖 R04，可并行 | 工作台、AI、沟通、健康材料、高风险会诊。 |
| R40 | 家长端 | 依赖 R04，可并行 | 首页、AI、反馈、绘本、提醒、移动端。 |
| R50 | 共享业务页 | 依赖 R20/R30/R40 收敛 | 儿童、健康、成长、饮食、弹窗/空态/权限态。 |
| R90 | 视觉 QA | 串行收口 | Playwright 截图、pixel diff、差异报告、功能门禁。 |
| R99 | 最终验收 | 串行最后 | 汇总所有 pass/fail、剩余风险、上线前决策。 |

## Parallel Rules
- R01/R02/R03 可并行。
- R04 依赖 R03。
- R10/R20/R30/R40 在 R04 后可并行，写集必须分离。
- R05/R06 可与角色页并行，但最终必须接入对应路由。
- R90/R99 必须串行执行。
