# Acceptance Criteria

每个补齐功能至少满足以下标准：

1. 页面可进入。
2. 主按钮有真实行为。
3. 提交、保存、回复后状态变化。
4. 刷新页面后数据仍存在。
5. 换相关角色后能看到对应数据，若业务应同步。
6. 不相关角色不能看到不属于自己的数据。
7. 空状态真实，不被 mock 数据覆盖。
8. 如果仍然只是 visual-only，必须明确禁用或标识。
9. 没有 fake success。
10. Browser Use 或 Playwright 可验证。

## D02 家园沟通验收

- 家长提交唯一 token 消息。
- 教师端同 child/class 可见。
- 教师回复后刷新仍可见。
- 家长端刷新或重新登录后可见回复。
- 其他班级教师不可见。
- 园长端可看到汇总和处理状态。

## D03 教师记录验收

- 晨检、饮食、成长记录可新增。
- 保存后刷新仍存在。
- 编辑后刷新仍保留更新。
- 家长端只读可见对应 child 记录。
- 保存失败时不显示成功态。

## D04 家长端验收

- `child` 参数不丢失。
- 未授权 child 不渲染、不提交。
- 成长档案、健康、餐谱、提醒使用同一 child-scoped 数据。
- 绘本保存/分享/导出未实现时必须禁用或标识。

## D05 健康材料和会诊验收

- 文件选择后有任务状态。
- 解析结果保存进 snapshot。
- 会诊可创建、查看、备注、更新状态。
- 教师端和园长端看到同一会诊不同视角。
- 高风险同步提示必须对应真实写入结果。

## D06 园长端验收

- 看板指标从当前 snapshot 聚合。
- AI 建议基于当前数据，fallback 可追踪。
- 周报/报表基于记录生成。
- 家长反馈和风险项处理后刷新保留。

## D07 共享操作验收

- 所有可点击按钮都有行为、禁用态或说明。
- fake-success 被替换为真实写入结果。
- visual-only 模块有明确标识或被真实功能替换。
- 表单失败状态可见。

## D08 测试验收

- Playwright 能验证消息闭环。
- Playwright 能验证记录刷新持久化。
- Playwright 能验证 child 参数隔离。
- Playwright 能验证 fake-success 失败态。
- bugbash smoke 结果被记录，不伪装失败为通过。
