# Pixel Acceptance Criteria

This phase is judged by whether pages clearly look like the GPT Image 2 design references, not by whether the previous design system merely feels consistent.

## Score Targets

- Critical pages: visual closeness >= 95.
- High-priority pages: visual closeness >= 90.
- Medium-priority pages: visual closeness >= 85.

## Required Evidence Per Page

Every page must have:

- Current screenshot before changes.
- Reference design image from `C:\Users\12804\Desktop\childcare-smart源代码\前端重构`.
- Modified screenshot after changes.
- Difference list.
- Fix list.
- Visual closeness score.
- Remaining gaps, if any.

## Required Fidelity

- Key visual elements shown in the design should be implemented or cropped and reused where safe.
- Missing real business support is not a reason to omit a visible design module; use visual-only frontend placeholders when needed.
- Core interactive areas should be real HTML/CSS whenever feasible.
- Tables, forms, cards, navigation, buttons, dialogs, and empty states should visually match the chosen references.
- Responsive layouts should match the closest available desktop, tablet, or mobile design reference.

## Rejection Rules

- Do not claim completion after only changing tokens.
- Do not claim completion after only writing documentation.
- Do not turn a page into a single full-page static image.
- Do not break login, routes, demo accounts, or permissions.
- Do not leave obvious layout drift, overflow, clipped text, or incoherent overlap.
