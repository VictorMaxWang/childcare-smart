# 功能状态分类

所有线程必须为每个被审计功能选择一个 `featureStatus`。如果一个功能同时符合多个状态，优先选择对实现风险表达最准确、最严重的状态，并在 `notes` 补充次要状态。

## 状态定义

| status | 定义 | 典型证据 |
| --- | --- | --- |
| `complete` | 功能真实可用，接入真实数据或明确本地持久化，刷新后状态合理。 | 有真实 API 或持久化层；刷新、重新登录、角色隔离通过。 |
| `partial` | 部分接入，能展示或读取，但提交、更新、删除、上传、反馈等不完整。 | 有 GET 无写入；能生成建议但无法保存。 |
| `ui-only` | 只有前端界面，没有真实行为。 | 按钮无 onClick；表单无 submit；无网络请求。 |
| `mock-only` | 使用硬编码、mock、demo 数据，未接真实数据源。 | `lib/mock`、fixture、demoSeed、静态数组驱动主数据。 |
| `visual-only` | 视觉复刻模块，只用于展示，没有真实业务逻辑。 | pixel replica 组件、固定卡片、无交互和数据层。 |
| `fake-success` | 点击后显示成功，但没有真实提交、没有持久化，刷新后丢失。 | toast 成功但无 API；刷新状态回退。 |
| `not-persisted` | 能改前端状态，但刷新或重新登录后丢失。 | React state/local 临时缓存，不写入服务端或明确持久层。 |
| `backend-missing` | 前端想调用功能，但缺少后端接口或接口未实现。 | 404/500/503；Brain proxy fallback；接口只有占位。 |
| `permission-incomplete` | 功能存在，但角色、孩子、班级、数据范围不正确。 | 跨账号可见；childId/classId 硬编码；教师看到错误班级。 |
| `broken` | 功能本应可用，但当前报错、白屏、按钮无反应、接口失败。 | console/page error；按钮无响应；关键请求失败。 |
| `needs-product-spec` | 设计图里有，但产品规则不明确，无法判断应该如何实现。 | 缺少业务规则、权限边界、字段定义或流转规则。 |

## 严重程度

| severity | 定义 |
| --- | --- |
| `F0` | 主流程完全不可用；数据提交到错误对象；跨角色/跨孩子数据风险；严重 fake success；导致数据丢失或错误持久化。 |
| `F1` | 核心功能只是 UI，没有实际接入；家园沟通、反馈、健康材料解析、成长记录等关键功能不可用；主要按钮无真实效果。 |
| `F2` | 功能部分可用，但缺少保存、编辑、刷新持久化、错误态、空态；mock 数据覆盖真实数据；visual-only 没有清晰标识。 |
| `F3` | 次要功能缺口；体验不完整；缺少 loading/error/empty 状态；缺少自动化测试。 |
| `F4` | 纯优化项；文案、提示、轻微体验问题。 |
