# 实现路线图

本文件由 C99 汇总线程维护。C00 只定义排序规则。

## 排序规则

1. 先处理 F0：数据错投、跨角色/跨孩子风险、严重 fake success、主流程完全不可用。
2. 再处理 F1：家园沟通、健康材料解析、高风险会诊、成长记录、周报等核心功能只做 UI 或无真实接入。
3. F2 按依赖排序：先补后端/数据模型，再补前端提交和持久化，再补空态/错误态。
4. F3/F4 放入收尾批次，避免阻塞核心功能闭环。
