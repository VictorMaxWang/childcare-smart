# 比赛统一口径指南

更新基准：`2026-05-31`

本文是国赛、校赛、机器人及人工智能大赛材料的口径锁定文档。后续 README、答辩稿、系统导览、截图包和录屏脚本如有冲突，以本文为准。

## 1. 标准命名

- 中文展示名：`慧育童行`
- 英文名 / 技术系统名：`SmartChildcare Agent`
- 推荐标题：`慧育童行 - SmartChildcare Agent`
- 推荐副标题：`面向托育场景的多智能体闭环决策系统`
- 项目一句话：`慧育童行（SmartChildcare Agent）把教师记录、AI 会诊、园长决策、家长理解和执行反馈组织成一条可演示、可复验、可继续生产化的托育智能体闭环。`

## 2. 比赛与赛道表述

- 可写：`机器人及人工智能大赛`、`校赛 / 国赛材料沿用同一产品口径`。
- 可写：`人工智能创新方向`、`多智能体协同决策系统`。
- 谨慎写：具体届次、赛道全称、奖项、承办单位，必须以报名系统或主办方通知为准。
- 禁止写：未确认的赛道名称、未确认获奖结果、未确认主办方背书。

## 3. 能力边界

- 可写：`vivo 能力已具备代码层接入、smoke/test 与受控演示验证。`
- 可写：`Chat/OCR/ASR 在当前演示发布链路中已有验证记录，provider 状态可通过工程材料核验。`
- 可写：`真实机构生产化仍需补齐对象存储、长期监控、正式审计、SLA 和完整隐私合规流程。`
- 禁止写：`fully live`、`真实机构已全量生产上线`、`完整生产化闭环已完成`、`所有上游能力已长期稳定运行`。

## 4. 主演示路线

答辩、录屏、截图和系统导览优先使用以下顺序：

1. `/login`：品牌首屏、示例账号、系统导览入口。
2. `/teacher`：教师工作台、语音或草稿入口。
3. `/teacher/high-risk-consultation`：高风险会诊过程、证据链、干预卡。
4. `/admin`：园长首页风险优先级、会诊承接、治理区、周报预览。
5. `/parent`：家长首页今晚行动、趋势入口、关怀模式、反馈入口。
6. `/parent/storybook?child=c-1`：成长微绘本。
7. `/parent/agent?child=c-1`：趋势追问与结构化反馈。

`/teacher/agent` 和 `/admin/agent?action=weekly-report` 只作为补充演示或备份路线，不再作为首选答辩主线。

## 5. 系统导览与视觉资产

- 系统导览中出现的 `慧育童行` 是中文展示品牌名。
- `SmartChildcare Agent` 是英文名、技术系统名和仓库主叙事名。
- 旧截图包或视觉稿中的 `智慧托育平台`、`普惠托育智慧管理平台` 只可视为历史产品描述，不再作为比赛主标题。
- 后续系统导览 PDF、截图包、海报和演示视频需统一使用 `慧育童行 - SmartChildcare Agent`。

## 6. 推荐截图清单

- 登录页：品牌首屏、示例账号入口、系统导览入口。
- 系统导览：封面、项目命名页、三端六能力页、总结页。

- Admin：园长首页风险优先级、会诊承接详情、周报 / 治理区、派单 / 跟进动作。
